# PC Parts Finder

Приложение на Python (Kivy/KivyMD) для поиска компьютерных комплектующих
сразу на нескольких польских сайтах: Media Expert, Media Markt, Neonet,
RTV Euro AGD. Оранжево-белый дизайн, фильтры (цена, состояние, магазин,
наличие, сортировка), карточки товаров.

---

## 1. Структура проекта

```
pcparts_app/
├── main.py              ← основная логика приложения
├── ui.kv                ← весь дизайн интерфейса (шапка, фильтры, карточки)
├── buildozer.spec        ← конфиг для сборки APK
├── requirements.txt      ← зависимости для запуска на компьютере
├── parsers/
│   ├── base.py            ← общий код парсера + функция очистки цены
│   ├── media_expert.py    ← парсер Media Expert
│   ├── media_markt.py     ← парсер Media Markt
│   ├── neonet.py          ← парсер Neonet
│   ├── rtv_euro_agd.py    ← парсер RTV Euro AGD
│   └── aggregator.py      ← запускает все парсеры параллельно
└── assets/icons/
    ├── app_icon.png       ← иконка приложения (сгенерирована)
    └── *.svg              ← ЗАГЛУШКИ логотипов магазинов (замени на реальные!)
```

## 2. САМОЕ ВАЖНОЕ — прочитай перед тем, как расстраиваться

Парсеры (`parsers/*.py`) написаны с **примерными** CSS-селекторами
(`div.product-tile`, `.price` и т.д.). Я не могу гарантировать, что они
сразу заработают на 100%, потому что:

1. У меня нет доступа к интернету, чтобы прямо сейчас открыть эти 4 сайта
   и списать точную структуру их HTML.
2. Магазины часто меняют вёрстку без предупреждения.
3. Некоторые страницы (особенно Media Markt) подгружают товары через
   JavaScript уже ПОСЛЕ загрузки страницы — тогда `requests` увидит
   пустой список, потому что видит только "сырой" HTML без JS-рендера.

**Что делать, если парсер вернул пустой список:**

1. Открой сайт магазина в браузере на компьютере → сделай поиск товара
2. Нажми `F12` (или Cmd+Opt+I на Mac) → вкладка "Elements"/"Инспектор"
3. Наведи на карточку товара → посмотри какой у неё класс (`class="..."`)
4. Открой `parsers/нужный_сайт.py` и поправь строки `soup.select(...)`
   под реальные классы
5. Если товаров в HTML вообще нет (пустой div, а данные явно должны
   быть) — значит сайт грузит их через JS. Тогда:
   - Попробуй в "Elements" найти `<script id="__NEXT_DATA__">` или
     `<script type="application/ld+json">` — там часто дублируются
     данные о товарах в виде обычного текста (JSON), доступного даже
     без выполнения JavaScript
   - Если и там нет — придётся переходить на `playwright` вместо
     `requests` (я могу помочь с этим отдельно, когда понадобится)

Пришли мне то, что видишь в "Elements" на реальном сайте — и я перепишу
селекторы под конкретную структуру.

## 3. Как протестировать на компьютере (ДО сборки APK)

Так удобнее всего отлаживать парсеры — быстрее, чем каждый раз собирать APK.

### Windows / Mac / Linux:

```bash
# 1. Установи Python 3.10-3.11 (с сайта python.org), если ещё нет
# 2. Перейди в папку проекта
cd pcparts_app

# 3. Создай виртуальное окружение (рекомендуется)
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Mac/Linux

# 4. Установи зависимости
pip install -r requirements.txt

# 5. Запусти
python main.py
```

Откроется окно с приложением — можно вводить запрос, тестировать фильтры,
и сразу видеть, работает ли парсинг (смотри в консоль — там будут
печататься ошибки парсеров, если что-то не так).

## 4. Как собрать APK (нужен компьютер с Linux)

Buildozer (инструмент сборки) работает **только на Linux**.
Если у тебя Windows — проще всего через **WSL** (Windows Subsystem for Linux).

### Шаг 1 — подготовка Linux-окружения

**На Windows:**
```powershell
wsl --install
```
Перезагрузи компьютер, откроется Ubuntu — создай пользователя/пароль.

**На Mac/Linux** — можно сразу переходить к шагу 2 (или использовать
виртуалку с Ubuntu, если сборка не пойдёт нативно на Mac).

### Шаг 2 — установка зависимостей для сборки (внутри Ubuntu/WSL)

```bash
sudo apt update
sudo apt install -y python3-pip python3-venv git zip unzip openjdk-17-jdk \
    autoconf libtool pkg-config zlib1g-dev libncurses5-dev cmake libffi-dev \
    libssl-dev build-essential

pip3 install --upgrade buildozer cython
```

### Шаг 3 — перенос проекта в Linux/WSL

Скопируй папку `pcparts_app` внутрь Ubuntu/WSL, например через:
```bash
# если файлы на Windows-диске, из WSL они видны так:
cp -r /mnt/c/Users/ТВОЁ_ИМЯ/Downloads/pcparts_app ~/pcparts_app
cd ~/pcparts_app
```

### Шаг 4 — сборка

```bash
buildozer android debug
```

Первая сборка займёт **много времени** (20-60+ минут) — Buildozer сам
скачает Android SDK, NDK и всё необходимое. Просто жди, не прерывай.

Если появится ошибка вида "License not accepted" — выполни:
```bash
yes | buildozer android update
```

### Шаг 5 — готовый APK

После успешной сборки файл появится тут:
```
~/pcparts_app/bin/pcpartsfinder-0.1-arm64-v8a_armeabi-v7a-debug.apk
```

Перенеси его на телефон (через USB, Google Drive, Telegram "Избранное" —
как угодно) и открой файл на телефоне → разреши "Установка из
неизвестных источников" при первом запросе → Установить.

## 5. Частые проблемы при сборке

| Проблема | Решение |
|---|---|
| `Command failed: ./gradlew` | Обычно нехватка памяти — закрой другие программы, либо добавь в `buildozer.spec` `android.gradle_dependencies` не трогать, просто перезапусти `buildozer android debug` — иногда помогает со второго раза |
| Зависает на "Installing android sdk" | Проверь интернет-соединение, попробуй заново |
| `Aidl not found` / `SDK license` | `yes | buildozer android update` |
| APK ставится, но приложение вылетает | Смотри лог: `adb logcat` (нужен Android Debug Bridge) при подключенном телефоне через USB |
| Приложение не показывает интернет-данные на телефоне (хотя на ПК работало) | Проверь, что в `buildozer.spec` прописано `android.permissions = INTERNET` (уже прописано в этом проекте) |

## 6. Что можно улучшить дальше

- Заменить SVG-заглушки логотипов в `assets/icons/` на настоящие логотипы магазинов
- Добавить кэширование результатов (чтобы не долбить сайты каждый раз)
- Добавить категории (процессор/видеокарта/...) как реальный фильтр по

Если что-то не соберётся или парсер не найдёт товары — пришли мне
скриншот ошибки или кусок HTML страницы магазина, разберёмся вместе.
